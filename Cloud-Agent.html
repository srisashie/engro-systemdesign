Problem

Design an agent that sits in Cloud infrastructure and handles alerts from various APIs to be made available for analysis.

Understanding the problem:

  1. Alerts - A message that follows a convention.
  1. Alerts by APIs - Various APIs log alerts messages during their operations.
  1. Agent -
    1. A component that reads and parses alert messages logged by API.
    1. Pass on parsed data to a dashboard.

Possible Solution - Serverless Function

  1. A lambda function in cloud that is invoked every time an alert is logged by API.
    1. Upside -
      1. Serverless function has no overhead of maintenance.
      1. Easy to setup and the function itself can be written in any language.
    1. Downside -
      • Lack of Scalability - Serverless functions are not scalable because of hard limit on parallel processing
      • **Cold starts latency only in case when alerts are not posted for longer time

High Level Design:

  • APIs would post alert messages to Lambda function.
  • Posting alerts to lambda would be Asynchronous.
  • Messages go through an API gateway which in-turn forwards them to Lambda.
  • The function hosted in Lambda would parse the alert and post it to the dashboard.
